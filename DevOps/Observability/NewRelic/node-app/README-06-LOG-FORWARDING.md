# New Relic Log Forwarding with Winston

## Overview

This guide explains how to send application logs from a Node.js application to New Relic using Winston.

---

## Step 1: Create a Logger Using Winston

Install Winston:

```bash
npm install winston
```

Create a logger:

```javascript
import winston from 'winston';

const logger = winston.createLogger({
  level: 'info',
  transports: [
    new winston.transports.Console()
  ]
});

export default logger;
```

Use the logger in your application:

```javascript
import logger from './logger.js';

logger.info('Application started');
logger.error('An error occurred');
```

---

## Step 2: Enable New Relic Log Forwarding

When starting the application, enable log forwarding by setting:

```bash
NEW_RELIC_APPLICATION_LOGGING_FORWARDING_ENABLED=true
```

Example:

```bash
NEW_RELIC_APP_NAME=node-app \
NEW_RELIC_LICENSE_KEY=<YOUR_LICENSE_KEY> \
NEW_RELIC_APPLICATION_LOGGING_FORWARDING_ENABLED=true \
node --experimental-loader=newrelic/esm-loader.mjs index.js
```

Or in `package.json`:

```json
{
  "scripts": {
    "start": "NEW_RELIC_APP_NAME=node-app NEW_RELIC_LICENSE_KEY=<YOUR_LICENSE_KEY> NEW_RELIC_APPLICATION_LOGGING_FORWARDING_ENABLED=true node --experimental-loader=newrelic/esm-loader.mjs index.js"
  }
}
```

---

## Step 3: Verify Logs in New Relic

After the application starts and generates logs, navigate to:

```text
Logs → Search Logs
```

You should be able to see logs generated by Winston.

Example logs:

```text
INFO  Application started
ERROR An error occurred
```

---

## Result

By creating logs with Winston and enabling:

```bash
NEW_RELIC_APPLICATION_LOGGING_FORWARDING_ENABLED=true
```

New Relic automatically collects and forwards application logs, allowing them to be viewed and searched directly in the New Relic Logs UI.
